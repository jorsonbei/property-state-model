# Contributing

Contributions should preserve the project's state-first and fail-closed boundaries.

1. Create a focused branch.
2. Add or update deterministic cases for behavior changes.
3. Run `make check`.
4. Keep generated evidence, local model outputs, private source documents, and secrets out of Git.
5. Report safety and answer-quality results separately. A zero gated-risk count is not evidence of open-domain answer quality.

Pull requests should state the affected `Q/Omega/phi/Delta sigma/Pi/eta/B_sigma/Sigma+` boundary and include the relevant verification result.
